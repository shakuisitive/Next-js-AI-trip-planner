# AI Trip Planner - Design System Guide

**For Web Designer Implementation**

## Project Overview

You're designing an AI-powered trip planning application that combines intelligent technology with travel inspiration. The design should feel trustworthy, modern, and exciting - conveying both technical sophistication and wanderlust.

## Color Palette - Ready to Use

### Primary Colors (Your Main Toolkit)

**Primary Blue - "Horizon"**

- **HEX:** `#2563EB`
- **RGB:** `37, 99, 235`
- **Use for:** Main buttons, navigation active states, primary CTAs, links
- **When to use:** Any primary action or main brand element

**Secondary Blue - "Deep Ocean"**

- **HEX:** `#1E40AF`
- **RGB:** `30, 64, 175`
- **Use for:** Button hover states, headers, active navigation, emphasis
- **When to use:** Darker variation of primary blue needed

**Accent Orange - "Sunset"**

- **HEX:** `#F97316`
- **RGB:** `249, 115, 22`
- **Use for:** Secondary buttons, highlights, "Add Trip" buttons, warm accents
- **When to use:** Secondary actions, attention-grabbing elements

**Success Green - "Journey Complete"**

- **HEX:** `#059669`

## Design Rules to Follow

### Color Distribution (60-30-10 Rule)

- **60%** - Use neutrals (whites, light grays) for backgrounds and main areas
- **30%** - Use primary blue for main interactive elements
- **10%** - Use accent orange and other colors for highlights

### Accessibility Requirements

- All text must have sufficient contrast (tools like WebAIM contrast checker)
- Color combinations in this guide meet WCAG standards
- Never rely on color alone to convey information

### Consistency Guidelines

- Always use `#2563EB` for primary actions
- Always use `#F97316` for secondary/add actions
- Keep button styles consistent across all pages
- Use the same shadows and border-radius throughout
